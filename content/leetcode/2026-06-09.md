### 19. 删除链表的倒数第 N 个结点

#### 我问了什么

- 一趟扫描怎么做？
- 这题用到了数学知识吗，还是简单 trick？
- `fast` 先走 `n` 步、再用 `fast.Next != nil` 的版本对不对？

#### 解答沉淀

- 这题是快慢指针固定距离模型，不是复杂数学。
- `fast` 先走 `n` 步，之后 `fast` 和 `slow` 同步走。
- 当循环条件是 `fast.Next != nil` 时，最后 `slow` 会停在待删除节点的前一个节点。
- 删除操作关注的是前驱节点，因为需要改的是 `slow.Next`。

#### 学到的思想

- 倒数第 `n` 个节点可以通过“快慢指针保持固定距离”转成正向扫描。
- 删除节点时，不要只想着找到目标节点，还要找到目标节点的前驱。
- 删除头节点可能发生时，加 `dummy` 可以统一处理。

#### 需要回看

- 为什么 `fast.Next != nil` 能让 `slow` 停在前驱节点。
- 删除类链表题为什么优先考虑 `dummy`。

### 876. 链表的中间结点

#### 我问了什么

- 这题能不能直接连 LeetCode 提交？
- 用 `dummy`、`fast.Next != nil && fast.Next.Next != nil`，最后返回 `slow.Next` 的写法对不对？
- 什么时候该用 `dummy`？

#### 解答沉淀

- 这题只是找位置，不改链表结构，常规写法不需要 `dummy`。
- `slow` 从 `head` 出发，`fast` 从 `head` 出发，`slow` 一步、`fast` 两步，最后返回 `slow`。
- `dummy` 写法也正确，但因为起点提前了一格，所以返回的是 `slow.Next`。

#### 学到的思想

- 找中点是快慢指针速度差模型。
- `fast` 到尾时，`slow` 到中点。
- 起点、循环条件和返回值必须配套。

#### 需要回看

- 找中点的 `head` 起点模板。
- `dummy` 的使用边界：改结构用，不改结构通常不用。

### 142. 环形链表 II

#### 我问了什么

- 为什么相遇后让一个指针回 `head` 可行？
- “fast 比 slow 多走的距离，必然是若干圈环长”是什么意思？
- `a = k * (b + c) - b` 这个式子怎么理解？
- 进阶要求 `O(1)` 空间是什么意思？
- 用 `map` 是什么意思？
- `map` 里的 key 是 `ListNode` 的地址吗，value 存的是 `bool` 吗？
- 这题是不是需要一定的数学知识？

#### 解答沉淀

- 快慢指针第一次相遇时，`fast` 比 `slow` 多走的距离一定是环长的整数倍。
- 设 `a` 是 `head` 到环入口的距离，`b` 是入口到相遇点的距离，`c` 是相遇点继续走到入口的距离，环长就是 `b + c`。
- 相遇后可以推出 `a = (k - 1) * (b + c) + c`，意思是：从 `head` 到入口的距离，等价于从相遇点走若干圈后再走 `c` 步到入口。
- 所以一个指针从 `head` 出发，另一个指针从相遇点出发，两者每次都走一步，最终会在环入口相遇。
- 哈希表做法是记录访问过的节点，第一次再次遇到的节点就是环入口；Go 里的具体写法单独放到小拾谈记复盘。

#### 学到的思想

- 判环是快慢指针速度差模型。
- 找环入口多了一层环上距离推导：相遇点不是入口，但它和入口之间的距离可以和 `head` 到入口的距离对齐。
- 这题可以先把“相遇后一个回 `head`，两个一起走，交点就是入口”记成结论，再慢慢理解推导。

#### 需要回看

- 环入口推导里的 `a`、`b`、`c` 各自代表什么。
- 为什么 `fast` 和 `slow` 的路程差一定是环长整数倍。
- 哈希表做法和快慢指针做法的空间复杂度差异。
